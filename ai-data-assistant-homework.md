# AI Data Assistant

Нужно реализовать сервис, который позволяет задавать вопросы к базе данных на естественном языке.

Пользователь пишет текстовый запрос, например:

```text
Top 30 countries by revenue
```

Сервис должен с помощью LLM построить SQL-запрос, выполнить его на базе данных и вернуть результат по данным.

Важно не сделать идеальный BI-инструмент, а показать инженерный подход к построению LLM-сервиса для работы с базой данных.

## Требования

- Использовать LLM через OpenAI-compatible API `v1/chat/completions`.
- Должна быть возможность указать свой LLM endpoint, model и API key.
- Сервис должен подключаться к PostgreSQL.
- PostgreSQL должен запускаться в Docker вместе с сервисом.
- Сервис не должен падать при ошибках LLM или базы данных.
- Сервис должен запускаться через Docker.
- Проект должен быть предоставлен как публичный GitHub-репозиторий.
- В репозитории нужно приложить все агентские и инструкционные файлы, которые использовались во время разработки, например `AGENTS.md`, `.cursorrules`, `CLAUDE.md` и аналогичные.
- В README нужно описать:
  - как запустить сервис;
  - как подключить и инициализировать базу данных;
  - какие переменные окружения нужны;
  - примеры запросов;
  - кратко, как устроено решение.

## Данные

В репозитории должны быть синтетические данные и всё необходимое для инициализации PostgreSQL.

Данные должны быть достаточно сложными, чтобы по ним можно было задавать нетривиальные аналитические вопросы. Чем богаче и реалистичнее схема и данные, тем лучше.

Можно ориентироваться на пример PostgreSQL sample database:

https://neon.com/postgresql/getting-started/sample-database

Решение должно быть воспроизводимым: после клонирования репозитория должна быть возможность с нуля запустить сервис и базу данных локально и проверить работу решения.

Не обязательно поддерживать все возможные базы данных и SQL-диалекты. Достаточно PostgreSQL.
